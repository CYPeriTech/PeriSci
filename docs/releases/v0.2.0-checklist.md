# PeriSci v0.2.0 Release Checklist

> 版本定位：**架构冻结阶段（Architecture Freeze）**
> 核心目标：**执行边界成立、CLI 可运行、dataset 可导出、canonical case 闭环成立**

---

## 一、版本与仓库状态（Release Hygiene）

- 当前分支干净（无未提交更改）
- 所有提交信息清晰，无 `fix / tmp / wip / debug` 遗留
- 准备打的 tag 名称为 `v0.2.0`
- `CHANGELOG.md` 已记录 v0.2.0 主要变化
- `VERSIONING.md` 与 v0.2.x 版本语义一致
- README 中未承诺 solver 能力（仅描述平台骨架与接口）

---

## 二、架构与边界（Architecture Boundaries）

项目分层必须保持：

```
core
api
apps
cases
examples
python
tests
docs
```

必须满足：

- `core` **不依赖** `api / apps / python / examples`
- `api` **仅依赖**`core`
- `apps` 仅调用 `api`
- `examples` 仅调用 `api`
- `tests` 仅调用 `api` 或 CLI
- `python` **不引入新语义（no new semantics）**
- `python` 作为边界层，仅调用 CLI

三梁执行结构必须成立：

```
config-beam
run-beam
export-beam
```

对应关系：

- **Config Beam** → `config_schema` / config object（输入契约）
- **Run Beam** → `run_case(config) -> results`（纯执行闭环）
- **Export Beam** → `export_dataset(results, dataset_spec)`（输出与资产化）

约束：

- `config_schema` 必须作为唯一输入语义来源（no hidden inputs）
- `run_case` **不得写文件（no file-side effects）**
- `export_dataset` 是**唯一允许写 dataset/** 的边界
- `results` 仅存在于内存态，不得直接落盘
- `manifest` 必须由 `export_dataset` 生成并绑定 provenance

验证机制（非三梁结构）：

- `validate_case` 属于**治理与门禁机制（validation layer）**
- `validate_case` 只读取 dataset / manifest / 结构化结果
- `validate_case` 不参与执行与导出流程（no side effects）

---

## 三、CLI 工具链（Command Line Interface）

以下 CLI 必须存在：

```
perisci-run
perisci-export
perisci-validate
```

验证：

### perisci-run

支持：

```
perisci-run --config -
```

行为：

- 从 stdin 读取 config JSON
- 输出 run bundle JSON
- **不得写任何文件**

---

### perisci-export

支持：

```
perisci-export --bundle - --out <output_dir>
```

行为：

创建：

```
<output_dir>/dataset/
```

最小结构：

```
dataset/
  manifest.json
  config/
  results/
  code/
  version/
```

---

### perisci-validate

支持：

```
perisci-validate <dataset_manifest.json>
```

行为：

输出 JSON validation report：

```
{
  "ok": true/false,
  "issues": [...]
}
```

---

## 四、Dataset 结构与规范（Dataset Specification）

必须符合：

```
docs/specs/dataset-spec.md
```

最小 dataset 结构：

```
dataset/
 ├ manifest.json
 ├ config/
 │   └ config.json
 ├ results/
 ├ code/
 └ version/
     ├ dataset_version.txt
     └ export_tool_version.txt
```

manifest 必须包含：

```
manifest
manifest_version
dataset_version
status
provenance
code
results
```

---

## 五、Canonical Cases（标准算例资产）

仓库必须包含：

```
cases/
```

至少存在：

```
cases/case-00-minimal
```

包含：

```
input.json
expected.json
notes.md
README.md
```

`input.json` 可被：

```
perisci-run
```

正确消费。

至少包含：

```
examples/ex-01-minimal-run/input.json
```

## 七、Python 层（Non-authoritative Python Layer）

目录：

```
python/perisci/
```

必须满足：

- Python **不引入新语义**
- Python 仅作为 CLI 包装层

必须遵守：

```
python → CLI → api → core
```

禁止：

```
python → core
python → internal api
```

Python README 必须明确：

```
This layer is non-authoritative.
```
