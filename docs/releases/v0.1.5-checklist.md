# PeriSci v0.1.5 发布前 Checklist

> 版本定位：**骨架阶段（Scaffold / Bootstrap）**
>  核心目标：**结构成立、边界明确、最小闭环可复现**

------

## 一、版本与仓库状态（Release Hygiene）

-  当前分支干净（无未提交更改）
-  所有提交信息清晰，无 “fix / tmp / wip” 遗留
-  准备打的 tag 名称为 `v0.1.5`（不带前缀歧义）
-  未在 README / 文档中出现超出 v0.1.x 的能力承诺

------

## 二、目录结构与分层边界（Architecture Boundaries）

-  `core / api / examples / tests / docs` 分层清晰
-  `core` **不依赖** `api / examples / python / apps`
-  `api` 仅通过 `core` 提供能力
-  `examples / tests` **只 include api**，不直接 include core
-  `include/` 与 `src/` 职责分离明确
-  `python/`、`apps/` 仅为占位（`.gitkeep`），未引入隐性承诺

------

## 三、构建系统（CMake / Toolchain）

-  根目录 `CMakeLists.txt` 可独立配置（out-of-source）

-  推荐流程（Ninja 单配置）在 README 中明确

-  使用 Ninja + Debug 能成功：

  ```
  cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Debug
  cmake --build build
  ```

-  不依赖 IDE 专有行为（可纯命令行构建）

- Windows / Ninja / GCC 工具链已实际验证可用

- `compile_commands.json` 为构建产物，不入库，由 build 目录生成并供开发工具使用


------

## 四、最小可运行闭环（Minimal Runnable Loop）

-  `perisci_core` 能独立编译为库
-  `perisci_api` 能链接 `core` 并导出最小 API
-  `examples/hello` 能成功编译并运行
-  `perisci-hello` 输出内容与 README 描述一致
-  examples 只使用对外 API（无内部泄漏）

------

## 五、测试与质量门禁（Quality Gates）

-  `tests/test_build.cpp` 能编译、链接、运行
-  `ctest --test-dir build` 在单配置下通过
-  测试失败时能输出可诊断信息（`--output-on-failure`）
-  tests 定位清晰：验证**结构与链接**，不承诺数值正确性
-   `.gitignore` 中忽略的文件（如 build/、本地缓存等）不作为构建或测试流程的前提条件（CI 语义一致）

------

## 六、文档与制度（Docs & Governance）

-  `ARCHITECTURE.md` 已冻结（宪法级）
-  `ROADMAP.md` 与 v0.1.x 定位一致（非承诺）
-  `VERSIONING.md` 清晰说明 v0.x 语义
-  `docs/adr/README.md` 已存在并说明规则
-  `docs/adr/0001-*.md`（Ninja 单配置决策）已：
  -  状态为 Accepted
  -  与 README 内容一致
-  README 中构建/运行命令**可复制即跑**
-  specs 文档（consistency_matrix / stub_conventions / header_naming）之间无矛盾，且未形成隐性功能承诺

------

## 七、语义与命名一致性（Semantic Consistency）

-  C++ 源文件使用 `.cpp`，头文件使用 `.hpp`
-  对外头文件路径使用 `include/perisci/...`
-  命名空间与目录结构一致（`perisci::core`, `perisci::api`）
-  `stub` 文件明确为阶段性占位，不承诺实现

------

## 八、发布前最后确认（Final Sanity Check）

-  从一个**全新环境**（或新 clone）完整跑通：

  - configure → build → run → test

-  README 与 examples README 中无过期路径或命令

-  可以自信地对他人说：

  > “这是一个**结构成立、可运行、可复现的 v0.1.5 骨架版本**”
